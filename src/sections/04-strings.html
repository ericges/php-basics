Strings & Interpolation

Single vs Double Quotes

Double quotes — interpolation

$msg = "Hello!\nHow are you?";
echo $msg;
// Output: Hello! (+ newline)
//         How are you?

$name = 'Alice';
echo "Hi, $name!";
// Output: Hi, Alice!

// Curly braces also work
//   and prevent ambiguity
echo "Hi, {$name}!";
// Output: Hi, Alice!

Single quotes — literal

$msg = 'Hello\nHow are you?';
echo $msg;
// Output: Hello\nHow are you?
// (literal backslash-n)

$name = 'Alice';
echo 'Hi, $name!';
// Output: Hi, $name!
// (no interpolation)

Use single quotes for plain strings (slightly faster, clearer intent).
Use double quotes when you need interpolation or escape sequences like \n, \t.

Always wrap variables in curly braces {$var} inside double-quoted strings. It's unambiguous and prevents subtle bugs.

Curly Brace Interpolation

<?php

$user = 'Alice';
$items = ['apple', 'banana', 'cherry'];

// Always use curly braces — clear, unambiguous, consistent
echo "Hello, {$user}!";
echo "First item: {$items[0]}";

// Without braces, this would break:
echo "She knows {$user}s";   // ✓ "She knows Alices"
// echo "She knows $user‍s";  // ✗ looks for $users variable

// Works with object properties too
//   $userObject = new User('Alice');
//   echo "Hello, {$userObject->name}!";
// Output: Hello, Alice!
Always wrap variables in curly braces {$var} inside double-quoted strings. It's unambiguous and prevents subtle bugs.
