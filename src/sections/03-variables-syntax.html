Syntax, Variables & Data

Defining Variables

<?php

// Variables always start with $
$greeting = 'Hello';
$year     = 2025;
$pi       = 3.14159;
$active   = true;

// Variable names are case-sensitive
$Name = 'Alice';
$name = 'Bob';    // different variable!

// Must start with a letter or underscore, then letters/digits/underscores
$_count = 0;       // valid
// $3items = 'no'; // ✗ invalid — starts with a digit
Every statement ends with a semicolon ; — forgetting it is the most common beginner mistake!

Basic Types

Scalar Types

<?php

$name    = 'Alice';  // string
$age     = 30;       // int
$price   = 9.99;     // float (aka double)
$isAdmin = false;    // bool

Array & Null

<?php

$colors      = ['red', 'green', 'blue'];  // array
$middle_name = null;                      // null — explicitly "no value"

Calling Functions

<?php

// PHP has hundreds of built-in functions — call them by name
$length = strlen('Hello');     // 5
$upper  = strtoupper('hello'); // "HELLO"
$random = random_int(1, 100);  // e.g. 42

// Functions can be nested
$trimmed = strtolower(trim('  HELLO  ')); // "hello"

// Some functions modify variables in place (pass by reference)
$fruits = ['banana', 'apple', 'cherry'];
sort($fruits);
// $fruits is now ['apple', 'banana', 'cherry']

Find every built-in function at php.net/manual/en/funcref.php

Constants

<?php

// Preferred: const (compile-time, only literal values)
const MAX_RETRIES = 3;
const APP_NAME    = 'My App';

// Alternative: define() (runtime, can use expressions)
define('TAX_RATE', 0.19);
define('RANDOM_ID', random_int(1, 1000));

// const cannot use function calls:
// const RANDOM_ID = random_int(1, 1000); // ❌ Fatal error

echo APP_NAME;  // My App
echo RANDOM_ID; // e.g. 472
Name constants in UPPER_SNAKE_CASE. Prefer const for simple literal values; use define() when the value comes from a function or expression or the assignment itself is conditional.

Comments

<?php

// Single-line comment

# Also a single-line comment (less common)

/*
 * Multi-line / block comment.
 * Useful for longer explanations.
 */
Prefer // over # for single-line comments.
The # symbol originates from Unix shell scripting where it is meant for the shebang line (#!). Using // keeps your PHP code idiomatic and avoids confusion.
