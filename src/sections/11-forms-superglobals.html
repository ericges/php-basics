Forms & Superglobals

GET — Search Example

<!-- GET — data visible in URL -->
<form action="/search.php" method="GET">
    <input type="search" name="q" placeholder="Search...">
    <button type="submit">Go</button>
</form>
GET 
<?php // search.php

// $_GET['q'] comes from ?q=... in the URL
$query = $_GET['q'] ?? '';

if ($query !== '') {
    echo "You searched for: {$query}";
}

GET parameters appear in the URL — ideal for searches, filters, and bookmarkable pages.

POST — Login Example

<!-- POST — data in request body -->
<form action="/login.php" method="POST">
    <label for="email-input">E-Mail Address</label>
    <input name="email" id="email-input" type="email">
    <label for="pass-input">Password</label>
    <input name="password" id="pass-input" type="password">
    <button type="submit">Log In</button>
</form>
POST 
<?php // login.php

// $_POST contains data from the form body
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $email    = $_POST['email'] ?? '';
    $password = $_POST['password'] ?? '';

    // Always validate & sanitize!
    if ($email === '' || $password === '') {
        die('All fields are required.');
    }

    if ($password !== 'secret') {
        die('Invalid credentials.');
    }

    echo "Welcome back, {$email}!";
}

POST data is sent in the request body — use for sensitive data and state-changing actions.

Superglobals Overview

Variable Contains
$_GET Query string parameters (?key=val)
$_POST POST request body data
$_REQUEST Merged GET + POST + COOKIE (avoid — ambiguous)
$_SERVER Server & request info (REQUEST_METHOD, HTTP_HOST, …)
$_SESSION Session data (after session_start())
$_COOKIE Cookies sent by the browser
$_FILES Uploaded file information
$_ENV Environment variables

Superglobals are available in every scope — no global keyword needed.

Never Trust User Input

<?php

// filter_input — validate and sanitize in one step
$email = filter_input(INPUT_POST, 'email', FILTER_VALIDATE_EMAIL);

if ($email === false || $email === null) {
    die('Invalid email address.');
}

// htmlspecialchars — prevent XSS when outputting user data
$name = $_POST['name'] ?? '';
$safe = htmlspecialchars($name, ENT_QUOTES, 'UTF-8');

echo "<p>Welcome, {$safe}</p>";
Rule of thumb: validate input on the way in, escape output on the way out. Never echo raw user data into HTML.
